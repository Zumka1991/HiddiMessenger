# Hiddi production

Production использует отдельный Compose-проект, но подключается к внешней сети
`lucky_default`, в которой уже работает Caddy. Порт сервера наружу не
публикуется: единственная внешняя точка входа — HTTPS на
`hiddi.myaifriend.su`.

Перед первым запуском:

1. Создать DNS `A`: `hiddi` → `138.124.244.42`.
2. Поместить release-образ `hiddi-server:0.1.0-alpha.1` на VPS.
3. Создать `secrets/bootstrap_secret` длиной не менее 32 символов и дать доступ
   только root.
4. Добавить содержимое `Caddyfile` в конфигурацию существующего Caddy.
5. Запустить `docker compose up -d`, проверить `/health`, затем перезагрузить
   Caddy.

SQLite и зашифрованные вложения находятся в volume `hiddi_data`. Их необходимо
резервно копировать. На текущем VPS свободно немного места, поэтому до переноса
вложений в S3 следует включить мониторинг диска.
